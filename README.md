# ESP32-S3 监控网页

## 项目整体介绍

这套项目分成两部分：ESP32-S3 设备程序、Vue 网页监控页面。

以获取温度数据为例子：ESP32 连上 WiFi 后搭建简易网页服务，对外提供温湿度数据接口；Vue 页面直接访问接口，在页面上展示温度、湿度。
整套代码支持静态IP、动态IP两种模式，分享给别人不用改源码，只需要在页面填设备IP就能用，自带跨域处理，不会出现浏览器跨域报错。

## 一、ESP32-S3 程序使用说明（main.cpp）

### 1. 基础功能

1. 连接WiFi，支持两种IP获取方式，动态IP默认开启；
2. 自带跨域配置，浏览器前端直接访问接口不会拦截；
3. 提供 `/api/data` 接口，返回标准JSON格式温湿度；
4. 19、20引脚指示灯，WiFi连接过程会交替闪烁；
5. 可选开启串口打印，查看设备IP、联网状态。

### 2. 关键宏定义（切换IP模式、调试打印）

```cpp
//#define STATICIP    // 取消注释=开启静态固定IP；注释掉=自动获取动态IP
//#define dubge       // 取消注释=串口打印联网、IP调试信息
```

#### 模式1：动态IP（分享、换WiFi推荐用）

1. 把 `#define STATICIP` 注释掉；
2. 程序烧录完成，打开串口监视器，波特率115200；
3. 串口会打印设备当前局域网IP，把这个IP填到网页里就能连接。

#### 模式2：静态IP（自己长期固定设备使用）

1. 取消 `#define STATICIP` 前面注释；
2. 修改下面三段IP，网段必须和当前路由器/热点保持一致：

```cpp
IPAddress local_IP(192, 168, 248, 100);  // ESP32固定访问地址
IPAddress gateway(192, 168, 248, 122);   // 路由器/热点网关地址
IPAddress subnet(255, 255, 255, 0);      // 家用局域网掩码，一般不用修改
```

网段不对会导致设备连不上WiFi。

> 可以在动态IP模式下查看对应的网段进行相应的修改

### 3. WiFi账号设置（开源前必须处理）

```cpp
const char *SSID     = "你的热点名称";
const char *PASSWORD = "你的热点密码";
```

上传代码到GitHub前，不要填自己真实WiFi名称和密码，保留示例文字，使用者自行修改。

### 4. 接口地址说明

1. `http://设备IP/`
   普通GET访问，页面返回文字提示，用来测试设备服务是否正常运行。
2. `http://设备IP/api/data`
   获取温湿度数据，Vue页面专用接口，返回 `{"temperature":100,"humidity":105}`。

### 5. 硬件引脚说明

IO19、IO20：等待WiFi连接时两个灯交替闪烁；联网成功后停止闪烁。
如果需要真实传感器数据，修改 `getSensorData()` 函数，替换里面固定测试数值即可。

### 6. 编译烧录步骤

1. Arduino/PlatformIO打开代码，开发板选择ESP32-S3；
2. 修改WiFi名称、密码；
3. 根据需求开启或关闭静态IP、串口调试；
4. 编译上传，打开串口查看设备IP。

## 二、Vue网页前端使用说明（App.vue）

### 1. 页面功能

1. IP输入框：直接填写ESP32设备地址，不用修改代码、不用重新打包；
2. 状态色块区分：等待连接、连接成功、连接失败三种状态一目了然；
3. 温湿度单独卡片展示；
4. 连接失败自动提示错误原因，附带对应的解决办法；
5. 手动刷新按钮，请求过程中按钮锁定，防止重复点击。

### 2. 项目运行前置

项目基于Vite+Vue开发，vite.config.js内已经配置@路径别名，直接使用即可。
安装依赖与运行命令：

```bash
# 安装依赖
npm install
# 本地调试运行
npm run dev
# 打包静态文件，打包后dist文件夹可直接发给别人使用
npm run build
```

### 3. 两种使用方式

#### 本地开发调试

运行npm run dev打开网页，顶部输入完整设备地址，示例：`http://192.168.248.100`，点击切换IP并重试，页面自动拉取数据。
连接成功显示绿色提示，连接失败红色提示，附带排错方案。

#### 打包分发使用

执行打包命令后生成dist文件夹，不需要安装Node环境，任何人直接打开里面index.html就能用，仅需在页面输入ESP32设备IP。

### 4. 开源注意事项

1. 代码内移除本地私有IP配置文件，页面默认使用示例IP；
2. 本地使用直接在输入框填写真实设备地址，不用把IP硬写进代码；
3. vite代理配置已经全部注释，打包后不会出现接口转发失效问题；
4. 本地私有配置文件添加git忽略，不会上传至GitHub，不存在内网IP泄露问题。

### 5. 页面内置故障解决办法

网页请求失败会自动展示三条排查方案：

1. 确认ESP32代码完整烧录，代码内包含CORS跨域相关函数；
2. Chrome浏览器可开启禁用跨域模式调试；
3. 点击页面内链接，单独访问ESP32接口，判断设备服务是否正常。

## 三、整套项目完整操作流程

### 自用固定IP流程

1. ESP32代码打开静态IP宏定义，填写对应网段IP、网关；
2. 填入WiFi账号密码，烧录进ESP32；
3. Vue页面输入静态IP，后续无需重复修改，直接使用。

### 分享他人/更换WiFi流程

1. ESP32代码注释静态IP宏，使用者填写自身WiFi信息烧录；
2. 串口监视器查看ESP32自动分配的动态IP；
3. 打开Vue网页，填入查到的IP即可连接，全程不用修改任何代码文件。

## 四、常见问题排查

1. 网页提示连接失败
   - 手机、电脑和ESP32连接同一个WiFi/热点；
   - 打开串口确认ESP32联网成功，IP无报错；
   - 确认烧录的ESP32代码没有删除CORS相关函数。
2. 能访问根页面，但/api/data拿不到数据
   检查ESP32代码内路由配置是否完整。
3. 开启静态IP后设备连不上WiFi
   核对本地IP、网关、子网掩码网段是否和路由器一致。
