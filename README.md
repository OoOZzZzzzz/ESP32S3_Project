# ESP32-S3 温湿度网页监控系统使用文档



![整体效果](https://github.com/OoOZzZzzzz/ESP32S3_Project/blob/main/image/%E6%95%B4%E4%BD%93%E6%95%88%E6%9E%9C.png?raw=true)



## 整体介绍

整套工程分成两部分：ESP32-S3 设备程序、Vue 网页监控页面。

 以获取温度数据为例子：ESP32 连上 WiFi 后搭建简易网页服务，对外提供温湿度数据接口；Vue 页面直接访问接口，在页面上展示温度、湿度。 整套代码支持静态IP、动态IP两种模式，分享给别人不用改源码，只需要在页面填设备IP就能用，自带跨域处理，不会出现浏览器跨域报错。

目前工程代码做了标准化的文件夹分层管理，所有功能模块都封装成独立库文件，实现设备和网页双向数据交互；前端页面需要手动安装Chart图形库，安装完成后可以实时绘制温湿度变化曲线，同时支持网页下发指令远程控制开发板硬件，整套系统既能实时查看环境模拟数据，也能远程操控外设。

## 一、程序使用说明（main.cpp）

### 1. 基础功能

1. 连接WiFi，支持两种IP获取方式，动态IP默认开启；
2. 带跨域配置，浏览器前端直接访问接口不会拦截；
3. 提供 `/api/data` 接口，返回标准JSON格式温湿度；
4. 19、20引脚指示灯，WiFi连接过程会交替闪烁；
5. 可选开启串口打印，查看设备IP、联网状态；
6. 新增下行指令接收功能，网页发送控制命令后，设备解析指令并控制IO19引脚LED亮灭，作为硬件控制演示；
7. 温湿度数据不再使用固定数值，改用正弦函数生成动态变化的模拟数据，数据会随时间持续浮动，贴近真实传感器采集效果。

### 补充：工程文件存放规范

为了方便后期维护和新增功能，代码按照Arduino/PlatformIO标准工程目录划分存放：

1. IO.cpp、send.cpp、receive.cpp 以及配套头文件全部放在项目 `lib` 文件夹，作为独立功能库调用；
2. 统一汇总所有模块的总头文件 heardfile.h 放置在 `include` 文件夹；
3. 主文件 main.cpp 仅需要引入 heardfile.h 一个头文件，不用重复导入各个功能模块，主程序逻辑简洁清晰。

### 2. 关键宏定义（切换IP模式、调试打印）

```c++
//#define STATICIP    // 取消注释=开启静态固定IP；注释掉=自动获取动态IP
//#define dubge       // 取消注释=串口打印联网、IP调试信息
```

#### 模式1：动态IP（分享、换WiFi推荐用）

1. 把 `#define STATICIP` 注释掉；
2. 程序烧录完成，打开串口监视器，波特率115200；
3. 串口会打印设备当前局域网IP，把这个IP填到网页里就能连接。

#### 模式2：静态IP（固定设备使用）

1. 取消 `#define STATICIP` 前面注释；
2. 修改下面三段IP，网段必须和当前路由器/热点保持一致：

```c++
IPAddress local_IP(192, 168, 248, 100);  // ESP32固定访问地址
IPAddress gateway(192, 168, 248, 122);   // 路由器/热点网关地址
IPAddress subnet(255, 255, 255, 0);      // 家用局域网掩码，一般不用修改
```

网段不对会导致设备连不上WiFi。

> 可以在动态IP模式下查看对应的网段进行相应的修改

### 3. WiFi账号设置

配置文件位置在 lib/send.cpp 顶部：

```cpp
const char *SSID     = "你的热点名称";
const char *PASSWORD = "你的热点密码";
```

### 4. 接口地址说明

1. `http://设备IP/` 普通GET访问，页面返回文字提示，用来测试设备服务是否正常运行。
2. `http://设备IP/api/data` 获取温湿度数据，Vue页面专用接口，返回动态正弦模拟温湿度JSON，示例：`{"temperature":25.1,"humidity":48.7}`。
3. `http://设备IP/api/cmd` 新增POST控制接口，接收网页下发的文本指令，识别led_on、led_off两条指令，用来控制硬件LED。

### 5. 硬件引脚说明

IO19、IO20：等待WiFi连接时两个灯交替闪烁；联网成功后停止闪烁。 IO19额外作为受控LED，接收网页指令切换亮灭状态。 如果需要真实传感器数据，修改 lib/send.cpp 里的 `getSensorData()` 函数，替换里面正弦模拟数值即可；硬件控制部分可以沿用当前指令解析逻辑，后续拓展继电器、风扇等其他外设。

### 6. 编译烧录步骤

1. Arduino/PlatformIO打开代码，开发板选择ESP32-S3；
2. 修改 lib/send.cpp 内WiFi名称、密码；
3. 根据需求开启或关闭静态IP、串口调试；
4. 完整上传 lib、include 文件夹内所有库文件，编译上传程序；
5. 打开串口查看设备IP。

### 现有使用小问题

设备WiFi连接采用阻塞等待逻辑，上电后联网等待时间较长；网页第一次填入IP加载数据、渲染图表时速度偏慢，画面会有轻微卡顿，初次使用耐心等待片刻即可正常加载。

## 二、Vue网页前端使用说明（App.vue）

### 1. 页面功能

1. IP输入框：直接填写ESP32设备地址，不用修改代码、不用重新打包；![IP](image\IP填写.png)
2. 状态色块区分：等待连接、连接成功、连接失败三种状态一目了然；
3. 温湿度单独卡片展示实时数值；![data](image\模拟数据.png)
4. 页面图表依赖Chart.js图形库，安装完成后会渲染温湿度实时波形曲线，温度、湿度分别使用左右双坐标轴展示，页面最多缓存30组历史数据，直观查看数据变化趋势；![picture](D:\Githup_workplace\ESP32S3_Project\image\检查图.png)
5. 硬件LED控制区域，页面提供点亮、关闭LED两个按钮，点击即可向ESP32发送控制指令；![LED](image\发送数据.png)
6. 连接失败自动提示错误原因，附带对应的解决办法；
7. 手动刷新按钮，请求过程中按钮锁定，防止重复点击；
8. 出现问题时曲线重置按钮，一键清空所有历史数据和波形，重新初始化图表。

### 2. 项目运行前置

项目基于Vite+Vue开发，vite.config.js内已经配置@路径别名，直接使用即可。 项目本身没有内置Chart图形依赖，下载代码后需要手动安装Chart.js相关库，完整执行命令：

```
# 安装项目基础依赖
npm install
# 单独安装图表可视化库Chart.js
npm install chart.js
# 本地调试运行
npm run dev
# 打包静态文件，打包后dist文件夹可直接发给别人使用
npm run build
```

### 3. 两种使用方式

#### 本地开发调试

1. 先执行npm install安装基础依赖，再执行npm install chart.js安装图形库；
2. 运行npm run dev打开网页，顶部输入完整设备地址，示例：`http://192.168.248.100`，点击切换IP并重试，页面自动拉取数据、绘制波形曲线。 连接成功显示绿色提示，连接失败红色提示，附带排错方案。首次加载图表会有卡顿，等待几秒波形就能正常渲染。

#### 打包分发使用

执行打包命令后生成dist文件夹，不需要安装Node环境，任何人直接打开里面index.html就能用，仅需在页面输入ESP32设备IP。dist是打包后的静态文件，已整合Chart库资源，其他人使用不用再次安装依赖。

### 4. 页面内置故障解决办法

网页请求失败会自动展示三条排查方案：

1. 确认ESP32代码完整烧录，lib库文件无缺失，代码内包含CORS跨域相关函数；
2. Chrome浏览器可开启禁用跨域模式调试；
3. 点击页面内链接，单独访问ESP32接口，判断设备服务是否正常。

### 常见报错补充

打开页面看不到波形、控制台报Chart相关报错： 本地开发环境未执行 `npm install chart.js` 安装图形库，执行安装命令后重启本地服务即可恢复。

## 三、整套项目完整操作流程

### 自用固定IP流程

1. ESP32代码打开静态IP宏定义，填写对应网段IP、网关；
2. 填入WiFi账号密码，烧录进ESP32；
3. Vue项目本地安装Chart图形库，运行开发服务，页面输入静态IP，后续无需重复修改，直接查看温湿度曲线、控制LED。

### 分享他人/更换WiFi流程

1. ESP32代码注释静态IP宏，使用者填写自身WiFi信息烧录；
2. 串口监视器查看ESP32自动分配的动态IP；
3. 接收网页源码的用户，需要先在本地终端执行npm install、npm install chart.js安装全部依赖，启动网页后填入查到的IP即可连接，全程不用修改任何代码文件。

## 四、常见问题排查

1. 网页提示连接失败
   - 手机、电脑和ESP32连接同一个WiFi/热点；
   - 打开串口确认ESP32联网成功，IP无报错；
   - 确认烧录完整lib、include所有库文件，没有删除CORS相关函数。
2. 能访问根页面，但/api/data拿不到数据 检查ESP32 send.cpp内路由配置是否完整。
3. 开启静态IP后设备连不上WiFi 核对本地IP、网关、子网掩码网段是否和路由器一致。
4. 点击LED控制按钮无响应、提示指令下发失败 确认ESP32已经正常联网，串口查看是否收到前端下发的指令。
5. 网页打开后曲线长时间不显示、页面卡顿 分两种情况：一是未安装chart.js图形库，控制台会抛出缺失依赖报错；二是当前程序固有情况，首次联网、图表初始化耗时较长，安装完依赖后等待一段时间即可恢复正常。
6. ESP32上电指示灯一直交替闪烁不停 WiFi账号密码填写错误、热点网段不匹配或者信号弱，设备无法接入网络。
